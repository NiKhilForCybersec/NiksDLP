Detection Rules

Detection rules define what content to look for and how to classify matches.

Rule Components

ComponentDescriptionExample
ConditionWhat to detectCredit Card >= 5
ExceptionWhen NOT to matchSender = finance@
SeverityImpact classificationHigh, Medium, Low, Info
ChannelsWhere rule appliesEmail, Web, Endpoint

Condition Types

  • Content Matches DCM — Pattern/keyword match
  • Content Matches IDM — Document fingerprint match
  • Content Matches EDM — Database record match
  • Content Matches VML — ML classifier match
  • File Type — Match specific file formats
  • File Size — Size threshold
  • Sender/Recipient — Email addresses, domains
  • Protocol — SMTP, HTTP, FTP, etc.
  • Endpoint User — User or group membership
  • URL Category — Web destination category
  • Application — Application generating traffic
  • Time — Business hours vs. off-hours

Combine conditions with logical operators:

Compound Condition Examples 
# High-risk external transfer
(Credit Card >= 5) 
  AND (Recipient Domain NOT = company.com)
  AND (Protocol = SMTP OR HTTP)

# After-hours sensitive data access
(Content Matches "Confidential" keyword)
  AND (Time = Outside Business Hours)
  AND (User NOT IN "After-Hours Authorized")

# Large file with PII
(File Size > 10MB)
  AND (SIN >= 1 OR Credit Card >= 1)
  AND (Destination = Cloud Storage)

Response Rule Configuration

Example Response Rules 
# Response Rule: High Severity - Block and Alert
Name: Block High Severity Incidents
Condition: Severity = High
Actions:
  - Block transmission
  - Send email notification to:
    - Sender (user notification template)
    - Manager (manager notification template)
    - DLP Team ([email protected])
  - Generate incident with full content capture
  - Set incident status: New
  - Assign to: DLP Review Queue

# Response Rule: Medium Severity - Warn User
Name: Warn on Medium Severity
Condition: Severity = Medium
Actions:
  - Display user notification popup
  - Allow with justification required
  - Generate incident
  - Send daily digest to DLP team

# Response Rule: Low Severity - Audit Only
Name: Audit Low Severity
Condition: Severity = Low
Actions:
  - Generate incident (no content capture)
  - Add to weekly summary report

Template Tip

Templates are starting points. Always customize detection thresholds, severity levels, and response rules for your organization's risk tolerance.

Policy Best Practices

  • Start with audit mode — Run policies without blocking for 2-4 weeks to baseline
  • Use severity levels wisely — Reserve "High" for true data breach risks
  • Implement tiered responses — Low=log, Medium=warn, High=block
  • Minimize exceptions — Each exception is a potential data leak vector
  • Review regularly — Monthly policy review, quarterly exception audit
  • Document everything — Policy names, descriptions, and change history
  • Test before production — Use test detection servers and endpoints
