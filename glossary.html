DLP Glossary

Comprehensive glossary of Data Loss Prevention terminology, acronyms, and concepts.

A

Activity Explorer
Microsoft Purview tool that shows user activities on sensitive content, including DLP policy matches, label changes, and file operations.
Auto-Labeling
Automatic application of sensitivity labels to content based on sensitive information detection. Can be client-side (Office apps) or service-side (SharePoint/OneDrive at rest).

C

CASB (Cloud Access Security Broker)
Security solution that provides visibility and control over cloud application usage. Acts as intermediary between users and cloud services. Examples: Microsoft Defender for Cloud Apps, Symantec CloudSOC, Netskope.
Classification
Process of categorizing data based on sensitivity level, content type, or business value. Can be manual (user-applied) or automatic (system-detected).
Confidence Level
Measure of detection accuracy for sensitive information types. Levels: Low (65%), Medium (75%), High (85%). Higher confidence = fewer false positives.
Content Explorer
Microsoft Purview tool showing where sensitive content exists across Microsoft 365. Provides inventory of classified data by type and location.
Corroborative Evidence
Supporting keywords or patterns that increase confidence in sensitive data detection. Example: "Credit Card" near a 16-digit number.

D

Data at Rest
Data stored in repositories (file servers, databases, cloud storage) rather than actively being transmitted. Scanned by discovery tools.
Data in Motion
Data being transmitted across networks (email, web uploads, file transfers). Monitored by network DLP and email DLP.
Data in Use
Data actively being accessed or modified on endpoints. Monitored by endpoint DLP agents.
DCM (Described Content Matching)
Symantec detection method using patterns, keywords, and data identifiers to find sensitive content.
DKE (Double Key Encryption)
Microsoft encryption method where customer controls one encryption key. Microsoft cannot decrypt content even with legal order.
DLP (Data Loss Prevention)
Technology and processes to detect and prevent unauthorized transmission or exposure of sensitive data.

E

EDM (Exact Data Matching)
Detection method matching content against exact values from a database (employee IDs, customer records). Near-zero false positives.
Endpoint DLP
DLP protection on user devices monitoring activities like USB copy, print, clipboard, cloud upload.
Enforce Server
Central management server in Symantec DLP architecture. Hosts policies, database, and console.

F

False Positive
Incorrect detection where legitimate content is flagged as sensitive. Key metric for DLP effectiveness.
Fingerprinting
Creating unique hash signatures of sensitive documents for detection even when content is modified. Used in IDM.

I

ICAP (Internet Content Adaptation Protocol)
Protocol for integrating DLP with web proxies. Allows proxy to send content to DLP for inspection.
IDM (Indexed Document Matching)
Symantec detection method using fingerprints of sensitive documents. Detects content fragments even in modified form.
Incident
Record of a DLP policy violation including details about content, user, and action taken.

L

Luhn Algorithm
Checksum formula used to validate credit card numbers, SINs, and other identification numbers. Helps reduce false positives.

M

MDCA (Microsoft Defender for Cloud Apps)
Microsoft's CASB solution providing cloud app discovery, DLP integration, and session controls.
MDE (Microsoft Defender for Endpoint)
Endpoint protection platform. Provides foundation for Purview Endpoint DLP functionality.
MIP (Microsoft Information Protection)
Former name for Microsoft's classification and labeling technology. Now part of Microsoft Purview.

N

Network DLP
DLP monitoring network traffic (email, web, FTP) via TAP/SPAN ports or inline inspection.
Network Monitor
Symantec passive DLP component monitoring network traffic without blocking.
Network Prevent
Symantec inline DLP component that can block or modify traffic containing sensitive data.

O

OCR (Optical Character Recognition)
Technology to extract text from images for DLP inspection. Detects sensitive data in screenshots, scanned documents.
Override
User action to bypass a DLP block, typically requiring justification. Tracked for audit purposes.

P

PII (Personally Identifiable Information)
Information that can identify an individual: name, SSN/SIN, address, email, phone, etc.
PHI (Protected Health Information)
Health information protected under regulations like HIPAA. Includes medical records, insurance info, treatment data.
PIPEDA
Personal Information Protection and Electronic Documents Act. Canadian federal privacy law governing private sector data handling.
Policy
DLP configuration defining what sensitive data to detect and what actions to take. Contains rules with conditions and actions.
Policy Tip
Notification shown to users when content matches a DLP policy. Can warn or explain why action is blocked.

R

Retention Label
Microsoft Purview label defining how long content should be retained and what happens at end of retention period.
Rule
Component of DLP policy defining specific conditions (what to detect) and actions (what to do). Policies contain one or more rules.

S

SASE (Secure Access Service Edge)
Cloud architecture combining network security (including DLP) with WAN capabilities. Examples: Zscaler, Netskope.
Sensitivity Label
Classification tag applied to content indicating sensitivity level. Can enforce encryption, watermarks, access controls.
SIN (Social Insurance Number)
Canadian 9-digit identifier for tax, employment, and government services. Key PII type for Canadian DLP.
SIT (Sensitive Information Type)
Pattern definition for detecting specific sensitive data (credit cards, SSN, SIN). Combines regex, keywords, checksums.

T

Trainable Classifier
Machine learning model trained to recognize document types (resumes, contracts, source code) without explicit patterns.
True Positive
Correct detection where content actually contains sensitive data. Opposite of false positive.

U

Unified Audit Log
Microsoft 365 central audit log containing all user and admin activities including DLP events.

V

VML (Vector Machine Learning)
Symantec ML-based detection method trained on document samples to classify similar content.

W

Watermark
Visual marking (text or image) added to documents by sensitivity labels. Can be header, footer, or background.
Workload
Microsoft 365 service location for DLP: Exchange, SharePoint, OneDrive, Teams, Devices.
