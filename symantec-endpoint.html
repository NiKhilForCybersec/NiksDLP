Symantec Endpoint DLP

Symantec Endpoint DLP provides agent-based data protection for Windows, macOS, and Linux endpoints. The DLP Agent monitors user activities on sensitive content and enforces policies locally, even when disconnected from the Enforce Server.

Agent Deployment

Windows Agent Installation

Prerequisites

  • .NET Framework 4.7.2 or later
  • Local administrator rights for installation
  • Network connectivity to Enforce Server (port 8200)
  • Antivirus exclusions configured

Interactive Installation

Command Prompt 
:: Navigate to agent installer directory
cd C:\DLP_Agent_15.8

:: Run installer with basic parameters
msiexec /i "Symantec DLP Agent.msi" ^
    ENFORCE_SERVER=enforce.company.com ^
    ENFORCE_PORT=8200 ^
    /qn /l*v C:\Temp\dlp_install.log

:: Verify installation
sc query "Symantec DLP Agent"
reg query "HKLM\SOFTWARE\Symantec\DataLossPrevention\Agent"

Silent Installation with All Parameters

Command Prompt 
msiexec /i "Symantec DLP Agent.msi" ^
    ENFORCE_SERVER=enforce.company.com ^
    ENFORCE_PORT=8200 ^
    AGENT_GROUP="Windows Endpoints" ^
    INSTALL_OUTLOOK_PLUGIN=1 ^
    INSTALL_NOTES_PLUGIN=0 ^
    ENABLE_HTTPS_INSPECTION=1 ^
    STEALTH_MODE=0 ^
    REBOOT=ReallySuppress ^
    /qn /l*v C:\Temp\dlp_install.log

macOS Agent Installation

Prerequisites

  • macOS 10.15 or later
  • Administrator privileges
  • System Extension approval (MDM recommended)
  • Full Disk Access permission
macOS Security Requirements

Starting with macOS 10.15, System Extensions require user approval or MDM pre-approval. Without approval, the agent will have limited functionality.

Installation Steps

Bash 
# Mount the DMG
hdiutil attach SymantecDLPAgent.dmg

# Run the installer package
sudo installer -pkg "/Volumes/Symantec DLP Agent/SymantecDLPAgent.pkg" -target /

# Configure Enforce Server connection
sudo /opt/Symantec/DataLossPrevention/Agent/bin/edpactl config \
    --server enforce.company.com \
    --port 8200

# Start the agent
sudo /opt/Symantec/DataLossPrevention/Agent/bin/edpactl start

# Verify status
/opt/Symantec/DataLossPrevention/Agent/bin/edpactl status

MDM Profile for System Extension

MDM Profile (XML) 
<?xml version="1.0" encoding="UTF-8"?>
<plist version="1.0">
<dict>

</dict>
</plist>

Linux Agent Installation

Prerequisites

  • RHEL/CentOS 7+ or Ubuntu 18.04+
  • Kernel 3.10 or later
  • Root privileges
  • Required packages: glibc, libstdc++, openssl
Bash (RHEL/CentOS) 
# Install dependencies
sudo yum install -y glibc libstdc++ openssl

# Extract and install agent
tar -xzf SymantecDLPAgent-Linux-15.8.tar.gz
cd SymantecDLPAgent-Linux
sudo ./install.sh

# Configure connection
sudo /opt/Symantec/DataLossPrevention/Agent/bin/edpactl config \
    --server enforce.company.com \
    --port 8200

# Enable and start service
sudo systemctl enable edpa
sudo systemctl start edpa

# Check status
sudo systemctl status edpa
/opt/Symantec/DataLossPrevention/Agent/bin/edpactl status

Mass Deployment Options

SCCM/ConfigMgr

  1. Create application package with MSI and transform file
  2. Configure detection method (registry key or file existence)
  3. Deploy to device collection with required deployment type

Group Policy

  1. Place MSI on network share accessible by target computers
  2. Create GPO with Computer Configuration → Software Installation
  3. Add MSI package as Assigned installation
  4. Use MST transform file for configuration parameters

Intune (Windows)

  1. Convert MSI to .intunewin format using Content Prep Tool
  2. Create Win32 app in Intune admin center
  3. Configure install command with parameters
  4. Assign to device groups
PowerShell - SCCM Detection Script 
# Detection script for SCCM/Intune
$regPath = "HKLM:\SOFTWARE\Symantec\DataLossPrevention\Agent"
$minVersion = [version]"15.8.0.0"

try {
    $installed = Get-ItemProperty -Path $regPath -ErrorAction Stop
    $currentVersion = [version]$installed.Version
    
    if ($currentVersion -ge $minVersion) {
        Write-Host "Symantec DLP Agent $currentVersion is installed"
        exit 0
    }
} catch {
    # Not installed
    exit 1
}

Agent Configuration

Configuration File Location

Platform Configuration Path
Windows C:\Program Files\Symantec\DataLossPrevention\Agent\config\
macOS /opt/Symantec/DataLossPrevention/Agent/config/
Linux /opt/Symantec/DataLossPrevention/Agent/config/

Key Configuration Options

edpa.conf 
# Primary Enforce Server
EnforceServer=enforce.company.com
EnforcePort=8200

# Failover server (optional)
EnforceServerFailover=enforce-dr.company.com

# Communication interval (seconds)
HeartbeatInterval=300
PolicyCheckInterval=600

# SSL/TLS settings
UseSSL=true
ValidateServerCert=true
channels.conf 
# Enable/disable channels
EnableRemovableStorage=true
EnablePrinting=true
EnableClipboard=true
EnableScreenCapture=true
EnableEmailMonitoring=true
EnableBrowserMonitoring=true
EnableCloudApps=true

# HTTPS inspection
EnableHTTPSInspection=true
HTTPSInspectionPort=8443

# Excluded processes (comma-separated)
ExcludedProcesses=antivirus.exe,backup.exe

Stealth mode hides the DLP agent from users:

  • No system tray icon
  • Hidden from Add/Remove Programs
  • Service name obfuscated
  • Process name changed
Legal Consideration

Check local privacy laws before deploying in stealth mode. Many jurisdictions require employee notification of monitoring.

Troubleshooting

Common Issues

Issue Possible Cause Resolution
Agent offline in Enforce Network connectivity, firewall Check port 8200, run test-connection
Policies not applying Agent group mismatch, sync failure Verify agent group, force update-policies
High CPU usage Large file scanning, many policies Add file exclusions, optimize policies
HTTPS inspection failing Certificate trust, proxy conflict Install root CA, check proxy settings
USB not monitored Driver conflict, disabled channel Check channel config, reinstall agent
