Endpoint DLP Concepts

Endpoint DLP protects data on user devices—laptops, desktops, and sometimes servers. Unlike network DLP which monitors traffic at network boundaries, endpoint DLP operates at the OS level to detect and prevent data exfiltration through local channels.

Why Endpoint DLP?

With remote work and cloud adoption, the traditional network perimeter has dissolved. Endpoint DLP addresses scenarios that network DLP cannot:

  • USB/Removable Media — Copy to flash drives doesn't cross the network
  • Printing — Local and network printing from the endpoint
  • Encrypted Traffic — Inspect before encryption, not at the proxy
  • Off-Network Users — Protection when not on VPN
  • Application-Level Control — Block specific apps from accessing data

Key Components

Software installed on endpoints that monitors user activity. Can be a dedicated agent or shared with EDR (like Microsoft Defender).

Local copy of DLP policies enabling offline protection. Synchronized when connected to management server.

Kernel-level driver that intercepts file operations (copy, move, open) before they complete.
