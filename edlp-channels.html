Endpoint DLP Channels

Endpoint DLP monitors various "channels" through which sensitive data can leave an organization. Understanding each channel's capabilities and limitations is essential for comprehensive data protection.

Removable Storage (USB/External Drives)

The most common data exfiltration vector. Endpoint DLP monitors file copy operations to removable media.

  • USB Flash Drives — Most common, all capacity sizes
  • External Hard Drives — USB/eSATA connected
  • SD/MicroSD Cards — Via built-in or USB reader
  • Mobile Phones (MTP) — When connected as storage device
  • Optical Media — CD/DVD burning (limited support)
  • Encrypted Drives — BitLocker, vendor-encrypted USB
ActionDescription
AuditLog the activity, allow transfer
WarnShow policy tip, allow with justification
BlockPrevent file copy to device
EncryptAllow only to encrypted/approved devices

Most DLP solutions support device whitelisting:

  • By Hardware ID — Allow specific device serial numbers
  • By Vendor ID — Allow all devices from a manufacturer
  • By Encryption Status — Only allow encrypted USB
  • By User Group — IT admins can use any device
Example Device ID 
# USB Device Hardware ID Format
USB\VID_0781&PID_5583      # SanDisk Cruzer
USB\VID_0951&PID_1666      # Kingston DataTraveler

# Find device ID in Windows
Get-PnpDevice -Class DiskDrive | Select-Object InstanceId

Cloud Storage & Sync

Monitors file uploads to cloud storage services via sync clients and web interfaces.

Service Sync Client Web Upload Notes
OneDrive ✓ ✓ Native integration in Purview
Dropbox ✓ ✓ Monitor or block sync client
Google Drive ✓ ✓ Drive for Desktop client
Box ✓ ✓ Box Drive and Box Sync
iCloud Limited ✓ macOS native, harder to monitor
WeTransfer N/A ✓ Web-only service
Best Practice

Block personal cloud storage sync clients (Dropbox, personal Google Drive) while allowing corporate-approved services. Use CASB for additional cloud app visibility.

Printing

Monitors print operations to detect sensitive documents being printed.

Local Printing

  • Direct USB-connected printers
  • Print to file (PDF, XPS)
  • Virtual printers

Network Printing

  • Network-connected printers
  • Print servers
  • Cloud print services

Print Monitoring Challenges

  • Content Inspection — Print spool files can be inspected, but some applications bypass normal printing
  • Rendered Content — Document is rendered as image, limiting text-based detection
  • Screen Capture to Print — Screenshots printed bypass document detection

Email Clients

Monitors email composition and attachment handling in desktop email clients.

Email Client Support Level Coverage
Microsoft Outlook Full Body, attachments, recipients
Outlook Web (OWA) Full Via browser monitoring
Gmail Web Full Via browser monitoring
Thunderbird Limited Attachment monitoring only (some vendors)
Apple Mail Limited macOS support varies by vendor

Browser Upload & Paste

Monitors file uploads and content pasted into web browsers.

Supported Browsers

  • Microsoft Edge — Best support, especially with Purview
  • Google Chrome — Requires browser extension for full monitoring
  • Mozilla Firefox — Extension-based monitoring
  • Other Browsers — Often blocked as "unallowed browsers"

Monitored Activities

  • File Upload — Via file picker or drag-and-drop
  • Content Paste — Text pasted into web forms
  • Download (some vendors) — Sensitive files downloaded from web
Browser Extension Required

For Chrome and Firefox monitoring, deploy the DLP browser extension via GPO or Intune. Without the extension, upload detection may be limited.

Clipboard Monitoring

Monitors copy/paste operations between applications.

Typical Capabilities

  • Copy Detection — Sensitive content copied to clipboard
  • Paste Detection — Paste to unallowed applications
  • Cross-Application — Paste from Word to personal email
Purview Limitation

Microsoft Purview Endpoint DLP only audits clipboard activity—it cannot block clipboard operations. Other vendors like Symantec and Digital Guardian can block clipboard transfers.

Network Share (UNC Paths)

Monitors file copy operations to network shares and mapped drives.

  • UNC Paths — \\server\share\folder
  • Mapped Drives — Z:\folder (treated as network)
  • DFS Paths — \\domain\dfs\path

Common Policy Patterns

  • Allow copy to approved file servers
  • Block copy to personal NAS devices
  • Audit all network copy operations

Bluetooth Transfer

Monitors file transfer via Bluetooth to phones, headsets, and other devices.

Platform Bluetooth DLP Support
Windows Full — Block file transfer via OBEX
macOS Limited/None — System restrictions

Remote Desktop (RDP) Clipboard

Monitors clipboard and drive redirection in RDP sessions.

  • Clipboard Redirection — Copy/paste between local and remote
  • Drive Redirection — Access to local drives from remote session
  • File Transfer — Drag-and-drop files into RDP window
RDP Protection

For highest security, disable clipboard and drive redirection via Group Policy on RDP servers. DLP can provide an additional layer of monitoring.

Screen Capture

Monitors screenshot and screen recording activities.

  • Print Screen — Keyboard shortcut capture
  • Snipping Tool / Snip & Sketch — Windows built-in tools
  • Third-Party Tools — Snagit, Greenshot, LightShot
  • Screen Recording — OBS, Camtasia, game recording
Detection Limitations

Screen capture DLP typically audits but cannot prevent captures. Content in screenshots is an image, making text-based detection ineffective on the captured output.
