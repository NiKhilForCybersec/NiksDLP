Quick Start Guide

Get your first DLP policy running in Microsoft Purview in under 30 minutes. This guide walks through creating, testing, and deploying a basic data protection policy.

Prerequisites
  • Microsoft 365 E3/E5 or Microsoft 365 Business Premium license
  • Compliance Administrator or Global Administrator role
  • Access to compliance.microsoft.com

Step 1: Access the Compliance Portal

  1. Navigate to Compliance Portal

    Go to compliance.microsoft.com

  2. Sign in

    Use your administrator credentials

  3. Navigate to DLP

    Click Data loss prevention in the left navigation, then Policies

Step 2: Create Your First Policy

Option A: Use a Template (Recommended for Beginners)

  1. Click Create Policy

    Click the + Create policy button

  2. Choose a Template

    Select Financial → Canada Financial Data (or your preferred template)

  3. Name Your Policy

    Example: "Canada Financial Data Protection - Pilot"

  4. Choose Locations

    Start with limited scope:

    • ☑ Exchange email
    • ☑ SharePoint sites (select specific pilot sites)
    • ☑ OneDrive accounts (select specific pilot users)
    • ☐ Teams (add later)
    • ☐ Devices (add later)
  5. Review Default Rules

    Template includes pre-configured rules for credit card, bank account detection

  6. Set Mode to Test

    Select Test it out first (test mode with notifications)

  7. Create Policy

    Click Submit to create the policy

Option B: Create Custom Policy

PowerShell - Create First Policy 
# Connect to Security & Compliance
Connect-IPPSSession

# Create policy targeting pilot users
New-DlpCompliancePolicy -Name "Quick Start - PII Protection" `
    -ExchangeLocation "All" `
    -SharePointLocation "https://company.sharepoint.com/sites/pilot" `
    -OneDriveLocation "[email protected]", "[email protected]" `
    -Mode "TestWithNotifications"

# Add detection rule
New-DlpComplianceRule -Name "Detect Credit Cards" `
    -Policy "Quick Start - PII Protection" `
    -ContentContainsSensitiveInformation @(
        @{Name="Credit Card Number"; minCount="1"; confidenceLevel="High"}
    ) `
    -NotifyUser "LastModifier" `
    -NotifyPolicyTipCustomText "This document may contain credit card information."

Write-Host "Policy created! Check Activity Explorer in 1-2 hours for results." -ForegroundColor Green

Step 3: Test Your Policy

Create Test Content

Test Document Content 
TEST DOCUMENT - DLP Policy Verification
========================================

This document contains test credit card numbers for DLP testing.

Test Credit Cards (Luhn-valid test numbers):
- Visa: 4111 1111 1111 1111
- MasterCard: 5500 0000 0000 0004
- Amex: 3400 0000 0000 009

Test Canadian SIN (use fake/test numbers only):
- SIN: 046-454-286

Note: These are TEST numbers only. Never use real PII for testing.

Test Scenarios

ScenarioActionExpected Result
SharePoint Upload Upload test document to pilot SharePoint site Policy tip appears, activity logged
Email External Email test document to external address Policy tip in Outlook, activity logged
OneDrive Sync Save test document to OneDrive folder Activity logged (may take longer)

Step 4: Review Results

Check Activity Explorer

  1. Navigate to Activity Explorer

    compliance.microsoft.com → Data classification → Activity explorer

  2. Filter by Policy

    Filter by your policy name to see matches

  3. Review Details

    Click on activities to see user, file, and match details

Check DLP Alerts

  1. Navigate to DLP Alerts

    compliance.microsoft.com → Data loss prevention → Alerts

  2. Review Alerts

    Look for alerts generated by your test activities

Patience Required

DLP activities can take 1-24 hours to appear in Activity Explorer. Don't panic if you don't see results immediately.

Step 5: Enable Policy

After successful testing (recommend 1-2 weeks), enable the policy:

PowerShell - Enable Policy 
# Move from test to enforce mode
Set-DlpCompliancePolicy -Identity "Quick Start - PII Protection" -Mode "Enable"

# Verify status
Get-DlpCompliancePolicy -Identity "Quick Start - PII Protection" | 
    Select-Object Name, Mode, Enabled

Common Issues

  • Wait 1-24 hours for policy to propagate
  • Verify test content contains valid test numbers (Luhn-valid)
  • Check that locations include your test site/user
  • Verify policy is enabled (not disabled)
  • Increase confidence level from Medium to High
  • Increase minimum match count (e.g., require 3+ credit cards)
  • Add keyword requirements near matches
  • Exclude specific file types or locations
  • Ensure policy is in "Test with notifications" or "Enable" mode
  • Verify NotifyUser is configured in the rule
  • Check user is using supported Office version
  • Policy tips may take time to appear (wait and retry)
