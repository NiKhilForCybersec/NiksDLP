Sensitive Information Types (SITs)

Sensitive Information Types are pattern-based classifiers that identify sensitive data like credit card numbers, social security numbers, and health records. Microsoft Purview includes 300+ built-in SITs and supports creating custom types.

Creating Custom SITs

Using the Compliance Portal

  1. Navigate to Data Classification

    Go to compliance.microsoft.com → Data classification → Sensitive info types

  2. Create New SIT

    Click "Create sensitive info type" and provide name, description

  3. Define Primary Pattern

    Add regex pattern that identifies the sensitive data format

  4. Add Supporting Elements

    Add keywords, secondary patterns, or validators to improve accuracy

  5. Set Confidence Levels

    Define which elements are required for low/medium/high confidence

  6. Test and Publish

    Test with sample content, then publish the SIT

PowerShell Creation

PowerShell 
# Connect to Security & Compliance PowerShell
Connect-IPPSSession -UserPrincipalName [email protected]

# Example: Create custom Employee ID SIT
# Pattern: EMP followed by 6 digits (e.g., EMP123456)

# Define the regex pattern
$employeeIdPattern = @{
    Pattern = "EMP\d{6}"
    Name = "Employee ID Pattern"
}

# Define supporting keywords
$keywords = @{
    Keywords = @("employee", "emp id", "staff number", "worker id")
    Name = "Employee Keywords"
    MatchType = "Word"
}

# Create the SIT
New-DlpSensitiveInformationType `
    -Name "Contoso Employee ID" `
    -Description "Matches Contoso employee IDs in format EMP######" `
    -SensitiveInformationTypeRulePackage @"

  
    
    
  
  
    
      
        
        
      
      
        
      
    
    EMP\d{6}
    
      
        employee
        emp id
        staff number
        worker id
      
    
  

"@

# Verify creation
Get-DlpSensitiveInformationType -Identity "Contoso Employee ID"

Custom Ontario Health Card (OHIP) SIT

XML Rule Package 
<!-- Ontario Health Card (OHIP) Custom SIT -->
<!-- Format: XXXX-XXX-XXX or XXXXXXXXXX (10 digits) -->

<Entity id="GUID-HERE" patternsProximity="300" 
        recommendedConfidence="75" name="Ontario Health Card (OHIP)">
  
  <!-- High confidence: Pattern + Keywords -->
  <Pattern confidenceLevel="85">
    <IdMatch idRef="OHIP_Regex"/>
    <Match idRef="OHIP_Keywords"/>
  </Pattern>
  
  <!-- Medium confidence: Pattern only -->
  <Pattern confidenceLevel="65">
    <IdMatch idRef="OHIP_Regex"/>
  </Pattern>
  
</Entity>

<!-- OHIP Number: 10 digits, optionally formatted as XXXX-XXX-XXX -->
<Regex id="OHIP_Regex">\b\d{4}[-\s]?\d{3}[-\s]?\d{3}\b</Regex>

<!-- Supporting keywords -->
<Keyword id="OHIP_Keywords">
  <Group matchStyle="word">
    <Term>OHIP</Term>
    <Term>Ontario Health</Term>
    <Term>health card</Term>
    <Term>health insurance</Term>
    <Term>health number</Term>
    <Term>ontario health insurance plan</Term>
  </Group>
</Keyword>

Testing SITs

Test in Compliance Portal

  1. Go to Data classification → Sensitive info types
  2. Select your SIT and click "Test"
  3. Enter or upload test content
  4. Review matches and confidence levels

PowerShell Testing

PowerShell 
# Test SIT against sample content
$testContent = @"
Customer: John Smith
SIN: 046-454-286
Credit Card: 4532-0151-1283-0366
Employee ID: EMP123456
OHIP: 1234-567-890
"@

# Test specific SIT
Test-DataClassification -TextContent $testContent `
    -SensitiveInformationType "Credit Card Number"

# Test all SITs
$results = Test-DataClassification -TextContent $testContent

# Display results
$results | ForEach-Object {
    Write-Host "SIT: $($_.SensitiveType)" -ForegroundColor Cyan
    Write-Host "  Count: $($_.Count)"
    Write-Host "  Confidence: $($_.Confidence)"
    Write-Host "  Matches: $($_.SensitiveTypeMatches.Value -join ', ')"
}

Best Practices

SIT Best Practices
  • Start with built-in SITs — Customize only when needed
  • Use high confidence for blocking — Low confidence for auditing
  • Combine with keywords — Reduces false positives significantly
  • Test thoroughly — Use real-world sample data
  • Document custom SITs — Include regex explanation and test cases
  • Review regularly — Data formats change over time
