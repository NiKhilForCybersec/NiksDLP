Getting Started

Recommended Learning Path

Start with the fundamentals to understand core DLP concepts before diving into specific platforms.

  1. Understand DLP Basics — Learn about data states, classification methods, and why DLP matters. Read DLP Fundamentals →
  2. Learn Data Classification — Understand regex, keywords, fingerprinting, and ML-based detection. Read Data Classification →
  3. Review Architecture Patterns — See how DLP components work together across endpoints, network, and cloud. Read DLP Architecture →
  4. Map to Compliance — Connect DLP controls to GDPR, HIPAA, PCI-DSS, and PIPEDA requirements. Read Compliance Frameworks →
Pro Tip

Start with Endpoint DLP in monitor-only mode to understand data flow before enabling blocking actions.

  1. Verify Licensing — Ensure you have Microsoft 365 E5/A5/G5 or E5 Compliance add-on. Check Licensing Requirements →
  2. Onboard Devices — Deploy endpoint agents via Intune, GPO, or SCCM. Endpoint Onboarding Guide →
  3. Configure Sensitive Info Types — Set up built-in and custom classifiers for your data. Configure SITs →
  4. Create DLP Policies — Build policies with conditions, actions, and user notifications. Create Policies →
  5. Monitor with Activity Explorer — Review detections and tune for false positives. Monitor Activity →
Important

Plan for a coexistence period where both platforms run simultaneously before decommissioning Symantec.

  1. Inventory Current Policies — Document all Symantec policies, response rules, and exceptions. Policy Inventory Guide →
  2. Map Detection Methods — Convert DCM, IDM, EDM to Purview equivalents. Detection Mapping →
  3. Phased Rollout — Start with monitor mode, progress to notify, then block. Phased Approach →
  4. Validate Coverage — Ensure all critical data types are detected by Purview. Validation Checklist →
  5. Decommission Symantec — Remove agents and servers after successful migration. Decommissioning Guide →
