Triage Criteria

SeverityCriteriaResponse Time
CriticalConfirmed data breach, large volume, external recipient< 1 hour
HighSensitive PII/PHI, suspicious pattern, executive involved< 4 hours
MediumPolicy violation, internal recipient, single occurrence< 24 hours
LowMinor violation, known business process, likely FP< 72 hours

Investigation Steps

  1. Review Incident Details

    Examine detected content, user, timestamp, channel, and matched policy.

  2. Assess Context

    Is this a legitimate business process? Is the user authorized? What was the destination?

  3. Gather Evidence

    Screenshot incident, export logs, document chain of custody if escalation likely.

  4. Determine Root Cause

    User error, malicious intent, process gap, or false positive?

Remediation Actions

  • False Positive — Update detection rules, add exception, close incident
  • User Error — Provide training, update process documentation
  • Policy Violation — Manager notification, HR involvement per policy
  • Malicious Activity — Escalate to security team, legal, HR; preserve evidence
