Sensitivity Labels

Sensitivity labels classify and protect documents and emails based on their sensitivity level. Labels can apply encryption, watermarks, headers/footers, and access restrictions that persist with the content.

Creating Sensitivity Labels

Using the Compliance Portal

  1. Navigate to Labels

    Go to compliance.microsoft.com → Information protection → Labels

  2. Create Label

    Click "Create a label" and provide name, display name, description

  3. Define Scope

    Choose: Items (files/emails), Groups & sites, or Schematized data assets

  4. Configure Protection

    Set encryption, content marking, access controls

  5. Auto-labeling (Optional)

    Configure conditions for automatic label application

  6. Publish Label

    Create label policy to publish to users

PowerShell Label Creation

PowerShell 
# Connect to Security & Compliance PowerShell
Connect-IPPSSession -UserPrincipalName [email protected]

# Create parent label
New-Label -Name "Confidential" `
    -DisplayName "Confidential" `
    -Tooltip "Business data that could cause harm if disclosed" `
    -Comment "Parent label for confidential content"

# Create sub-label with encryption
New-Label -Name "Confidential-AllEmployees" `
    -DisplayName "Confidential \ All Employees" `
    -ParentId (Get-Label -Identity "Confidential").Guid `
    -Tooltip "Confidential - All employees can access" `
    -EncryptionEnabled $true `
    -EncryptionProtectionType "Template" `
    -EncryptionTemplateId "All Employees - Confidential" `
    -ContentMarkingHeaderEnabled $true `
    -ContentMarkingHeaderText "CONFIDENTIAL" `
    -ContentMarkingHeaderFontColor "#FF0000" `
    -ContentMarkingHeaderFontSize 12 `
    -ContentMarkingFooterEnabled $true `
    -ContentMarkingFooterText "This document is confidential. Do not distribute."

# Create sub-label for Finance
New-Label -Name "Confidential-Finance" `
    -DisplayName "Confidential \ Finance" `
    -ParentId (Get-Label -Identity "Confidential").Guid `
    -Tooltip "Confidential - Finance team only" `
    -EncryptionEnabled $true `
    -EncryptionProtectionType "UserDefined" `
    -EncryptionDoNotForward $false `
    -EncryptionPromptUser $true

# View created labels
Get-Label | Select-Object Name, DisplayName, ParentId, Priority

Encryption Settings

Assign Permissions Now (Admin-defined)

Administrator pre-defines who can access content with this label.

  • Specify users, groups, or domains
  • Choose permission level (Viewer, Reviewer, Co-Author, Co-Owner)
  • Set content expiration and offline access
Permission LevelRights Included
ViewerView, Open, Read
ReviewerView, Edit, Save (no Print)
Co-AuthorView, Edit, Save, Print, Copy
Co-OwnerAll rights including change permissions

Let Users Assign Permissions

Users choose recipients when applying the label in Outlook or Office apps.

  • Do Not Forward — Recipients can't forward, print, or copy
  • Encrypt-Only — Encrypts but allows all permissions
  • Custom — User selects recipients and permissions

Double Key Encryption (DKE)

For highest sensitivity data requiring customer-controlled keys.

  • Content encrypted with two keys
  • One key held by Microsoft, one by customer
  • Microsoft cannot decrypt without customer key
  • Requires additional infrastructure setup
DKE Consideration

DKE content cannot be processed by Microsoft services (search, eDiscovery, Copilot). Use only for data that must never leave your control.

Content Marking

PowerShell - Content Marking 
# Configure header
Set-Label -Identity "Highly-Confidential" `
    -ContentMarkingHeaderEnabled $true `
    -ContentMarkingHeaderText "HIGHLY CONFIDENTIAL" `
    -ContentMarkingHeaderFontName "Arial" `
    -ContentMarkingHeaderFontSize 14 `
    -ContentMarkingHeaderFontColor "#C41E3A" `
    -ContentMarkingHeaderAlignment "Center" `
    -ContentMarkingHeaderMargin 10

# Configure footer
Set-Label -Identity "Highly-Confidential" `
    -ContentMarkingFooterEnabled $true `
    -ContentMarkingFooterText "© Contoso Corp. Unauthorized disclosure prohibited." `
    -ContentMarkingFooterFontSize 10 `
    -ContentMarkingFooterFontColor "#666666" `
    -ContentMarkingFooterAlignment "Center"

# Configure watermark
Set-Label -Identity "Highly-Confidential" `
    -ContentMarkingWatermarkEnabled $true `
    -ContentMarkingWatermarkText "CONFIDENTIAL" `
    -ContentMarkingWatermarkFontSize 72 `
    -ContentMarkingWatermarkFontColor "#808080" `
    -ContentMarkingWatermarkLayout "Diagonal"

Auto-Labeling

Auto-labeling automatically applies labels based on content conditions.

Client-side Auto-Labeling

Runs in Office apps, recommends or applies labels as users work:

PowerShell - Auto-labeling Condition 
# Set auto-labeling for client (recommend)
Set-Label -Identity "Confidential-Finance" `
    -AutoApplyType "Recommend" `
    -SensitiveInformationTypes @(
        @{Name = "Credit Card Number"; minCount = 1},
        @{Name = "Canada Social Insurance Number"; minCount = 1}
    ) `
    -LocaleSettings @{
        LocaleKey = "DisplayName"
        Settings = @(
            @{Key = "en-us"; Value = "Recommended: Confidential - Finance"}
        )
    }

Service-side Auto-Labeling

Runs in SharePoint, OneDrive, Exchange to label content at rest:

PowerShell - Service Auto-labeling Policy 
# Create auto-labeling policy for SharePoint/OneDrive
New-AutoSensitivityLabelPolicy -Name "Auto-Label PII Documents" `
    -SharePointLocation "All" `
    -OneDriveLocation "All" `
    -ApplyContentLabel "Confidential-AllEmployees" `
    -Mode "TestWithoutNotifications"

# Create rule for the policy
New-AutoSensitivityLabelRule -Name "PII Detection Rule" `
    -Policy "Auto-Label PII Documents" `
    -ContentContainsSensitiveInformation @(
        @{Name = "Canada Social Insurance Number"; minCount = 1; confidenceLevel = "High"},
        @{Name = "Canada Passport Number"; minCount = 1; confidenceLevel = "High"}
    ) `
    -WorkLoad "SharePoint,OneDriveForBusiness"

# After testing, enable enforcement
Set-AutoSensitivityLabelPolicy -Identity "Auto-Label PII Documents" `
    -Mode "Enable"

Publishing Labels

Labels must be published via a label policy to be available to users:

PowerShell - Label Policy 
# Create label policy for all users
New-LabelPolicy -Name "Global Sensitivity Labels" `
    -Labels "Public","Internal","Confidential","Confidential-AllEmployees",
            "Confidential-Finance","Highly-Confidential" `
    -ExchangeLocation "All" `
    -SharePointLocation "All" `
    -OneDriveLocation "All" `
    -ModernGroupLocation "All" `
    -Comment "Standard sensitivity labels for all employees"

# Configure policy settings
Set-LabelPolicy -Identity "Global Sensitivity Labels" `
    -AdvancedSettings @{
        RequireDowngradeJustification = "True"
        OutlookDefaultLabel = "Internal"
        PowerBIDefaultLabel = "Internal"
    }

# Require label for documents
Set-LabelPolicy -Identity "Global Sensitivity Labels" `
    -MandatoryLabel $true `
    -MandatoryLabelJustificationRequired $true

# Check policy deployment status
Get-LabelPolicy | Select-Object Name, Mode, Workload, DistributionStatus

DLP Integration

Sensitivity labels work with DLP policies for enhanced protection:

PowerShell - DLP Policy Using Labels 
# DLP policy that triggers on sensitivity label
New-DlpComplianceRule -Name "Block External Sharing of Highly Confidential" `
    -Policy "External Sharing Protection" `
    -ContentContainsSensitivityLabels "Highly-Confidential" `
    -BlockAccess $true `
    -BlockAccessScope "PerAnonymousUser" `
    -NotifyUser "SiteAdmin" `
    -GenerateAlert $true

# Combine label + SIT for higher confidence
New-DlpComplianceRule -Name "Protect Labeled PII" `
    -Policy "PII Protection" `
    -ContentContainsSensitivityLabels "Confidential-AllEmployees" `
    -ContentContainsSensitiveInformation @(
        @{Name = "Canada Social Insurance Number"; minCount = 1}
    ) `
    -BlockAccess $true `
    -NotifyUser "LastModifier"

Best Practices

Start Simple

Begin with 3-5 labels. Users won't use a complex taxonomy. Expand later based on needs.

Default Label

Set a default label (e.g., "Internal") so unlabeled content is rare.

Require Justification

Require justification when downgrading labels to track intentional changes.

Use Sub-labels

Create sub-labels for department-specific protection under common parent labels.
