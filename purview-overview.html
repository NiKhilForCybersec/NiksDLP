Microsoft Purview DLP Overview

Microsoft Purview Data Loss Prevention (DLP) is a cloud-native solution integrated with Microsoft 365 that helps organizations discover, classify, and protect sensitive information across endpoints, Microsoft 365 services, and cloud applications.

Rebranding Note

Microsoft Purview was formerly known as "Microsoft 365 Compliance" and "Microsoft Information Protection (MIP)". Documentation and UI may still reference these older names.

Key Capabilities

Discover

Identify sensitive data across Microsoft 365 workloads, endpoints, and on-premises file shares using 300+ built-in sensitive information types.

Classify

Apply sensitivity labels automatically or manually to documents and emails. Labels persist with content and can trigger protection policies.

Protect

Block, warn, or audit sensitive data transfers across email, cloud apps, USB drives, printing, and more with unified DLP policies.

Protected Locations

Purview DLP policies can protect sensitive data across these locations:

Location Protection Type Key Features
Exchange Online Email DLP Policy tips in Outlook, block/encrypt attachments, mail flow rules
SharePoint Online Cloud DLP Block sharing, restrict access, sensitivity labels
OneDrive for Business Cloud DLP Sync client integration, sharing controls
Microsoft Teams Chat & Channel DLP Block messages, policy tips in chat, file protection
Windows/macOS Endpoints Endpoint DLP USB, print, clipboard, browser upload, app restrictions
Power BI BI DLP Sensitivity labels on datasets and reports
On-premises file shares MIP Scanner Discovery and labeling of on-prem repositories
Cloud Apps (MCAS) Defender for Cloud Apps Third-party SaaS protection via CASB

Licensing Requirements

DLP features vary by license tier. Here's what's included at each level:

Microsoft 365 E3 / A3 / G3

Basic DLP capabilities for Exchange, SharePoint, and OneDrive:

  • ✓ DLP policies for Exchange Online
  • ✓ DLP policies for SharePoint Online
  • ✓ DLP policies for OneDrive for Business
  • ✓ 100+ built-in sensitive information types
  • ✓ Manual sensitivity labeling
  • ✓ Basic DLP reports
  • ✗ Endpoint DLP
  • ✗ Teams DLP
  • ✗ Auto-labeling
  • ✗ Trainable classifiers
  • ✗ Exact Data Match (EDM)

Microsoft 365 E5 / A5 / G5

Full DLP capabilities including Endpoint DLP and advanced features:

  • ✓ Everything in E3, plus:
  • ✓ Endpoint DLP (Windows/macOS)
  • ✓ Teams DLP (chat and channels)
  • ✓ Auto-labeling policies
  • ✓ Trainable classifiers (pre-trained and custom)
  • ✓ Exact Data Match (EDM)
  • ✓ Document fingerprinting
  • ✓ 300+ sensitive information types
  • ✓ Activity Explorer
  • ✓ Content Explorer
  • ✓ DLP Alerts dashboard
  • ✓ Microsoft Defender for Cloud Apps integration

E5 Compliance Add-on

Add E5 DLP features to E3 licenses:

  • Microsoft 365 E5 Compliance — Full compliance suite including DLP, eDiscovery, Insider Risk
  • Microsoft 365 E5 Information Protection & Governance — DLP, labeling, and data lifecycle management
Cost Optimization

If you only need Endpoint DLP (not full E5), the E5 Compliance add-on to E3 can be more cost-effective than upgrading all users to full E5.

Getting Started Checklist

  1. Verify Licensing

    Confirm you have E5 or E5 Compliance add-on for Endpoint DLP and advanced features.

  2. Assign Admin Roles

    Grant Compliance Administrator role to team members who will manage DLP.

  3. Enable Endpoint DLP

    Turn on device monitoring in Settings → Device onboarding.

  4. Onboard Devices

    Deploy MDE onboarding via Intune, GPO, or SCCM. See Endpoint DLP Guide

  5. Create Test Policy

    Start with a simple policy in Test mode to validate detection.

  6. Monitor Activity Explorer

    Review detected activities and tune policies before enabling enforcement.
