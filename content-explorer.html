Content Explorer

Content Explorer provides visibility into sensitive data across Microsoft 365, showing where classified content exists and how it's being used. It's essential for understanding your data landscape before implementing DLP policies.

Accessing Content Explorer

  1. Navigate to Compliance Portal

    Go to compliance.microsoft.com

  2. Open Data Classification

    Select Data classification from the left navigation

  3. Select Content Explorer

    Click on the Content explorer tab

Required Permissions

RolePermissionsUse Case
Content Explorer List Viewer View list of items and their classifications Compliance analysts reviewing data distribution
Content Explorer Content Viewer View actual content of classified items Investigators needing to see file contents
Data Classification List Viewer View classification statistics Reporting and dashboards
Data Classification Content Viewer Full access to content and metadata Full investigation capabilities
PowerShell - Assign Content Explorer Roles 
# Assign Content Explorer List Viewer role
Add-RoleGroupMember -Identity "Content Explorer List Viewer" -Member "[email protected]"

# Assign Content Explorer Content Viewer (includes list viewer)
Add-RoleGroupMember -Identity "Content Explorer Content Viewer" -Member "[email protected]"

# Create custom role group with specific permissions
New-RoleGroup -Name "DLP Analysts" -Roles "Data Classification List Viewer", "View-Only Audit Logs"
Add-RoleGroupMember -Identity "DLP Analysts" -Member "[email protected]"

Exporting Data

PowerShell - Export Content Explorer Data via Graph API 
# Connect to Microsoft Graph
Connect-MgGraph -Scopes "InformationProtectionPolicy.Read.All"

# Note: Direct Content Explorer export requires Graph API or manual export
# Use Activity Explorer for audit log export

# Alternative: Query via Compliance Search
$search = New-ComplianceSearch -Name "SIN Data Discovery" `
    -ExchangeLocation "All" `
    -SharePointLocation "All" `
    -ContentMatchQuery "SensitiveType:Canada Social Insurance Number"

Start-ComplianceSearch -Identity $search.Name

# Wait for completion then get results
Get-ComplianceSearch -Identity $search.Name | Select-Object Items, Size

Best Practices

Content Explorer Best Practices
  • Baseline first — Use Content Explorer to understand your data landscape before creating DLP policies
  • Focus on high-risk — Prioritize review of externally shared content with sensitive data
  • Regular audits — Schedule monthly reviews to track data proliferation trends
  • Limit content viewer access — Only grant content viewing to those who need it (privacy concern)
  • Combine with Activity Explorer — Use both tools for complete visibility
  • Document findings — Export and document baseline for compliance evidence
